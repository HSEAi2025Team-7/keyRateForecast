# Проект KeyRateForecast  

## 1. Цель проекта  
Создать прототип машинного обучения для анализа и классификации текстов публикаций Центрального Банка РФ.  

---

## 2. Данные  
- Источник: сайт ЦБ РФ (публикации и пресс-релизы).  
- Формат: таблица с колонками  
  - `text` — текст публикации,  
  - `label` — категория текста.  

---

## 3. Инструменты  
- **Python**  
- **Jupyter Notebook**  
- Библиотеки: `pandas`, `numpy`, `nltk`, `scikit-learn`, `matplotlib`  

---

## 4. Предобработка текстов  
- Привести к нижнему регистру.  
- Удалить ссылки, цифры, спецсимволы.  
- Удалить стоп-слова («и», «в», «на» и др.).  
- Лемматизация — приведение слов к начальной форме (например: «ставки» → «ставка»).  

---

## 5. Векторизация  
**Векторизация — перевод текста в набор чисел.**  
Используем метод **TF-IDF**:  
- TF (term frequency) — сколько раз слово встречается в тексте.  
- IDF (inverse document frequency) — насколько слово уникально для всех текстов.  

---

## 6. Модели машинного обучения  

1. **Логистическая регрессия (Logistic Regression)**  
   - Делит тексты по категориям.  

2. **Наивный Байес (Naive Bayes)**  
   - Используется для нахождения характерных слов, которые чаще всего встречаются в текстах каждой категории.  

3. **Случайный лес (Random Forest)**  
   - Состоит из множества простых деревьев решений.  
   - Итоговое решение получается голосованием всех деревьев.  

---

## 7. Метрики и валидация  
Данные делятся на 80% — обучение и 20% — проверку.  

Метрики:  
- **Accuracy (точность классификации)**  
- **Precision (точность для одного класса)**  
- **Recall (полнота)**  
- **F1-мера**  
- **Confusion matrix (матрица ошибок)**

## 9. Таймлайн  

### Ноябрь — подготовка данных и первые шаги  
- Собрать и сохранить данные (CSV).  
- Сделать разведочный анализ (EDA): длина текстов, частые слова, классы.  
- Реализовать очистку текстов (нижний регистр, стоп-слова, лемматизация).  

### Декабрь — векторизация и baseline  
- Сделать TF-IDF для текстов.  
- Построить первую модель (Logistic Regression).  
- Проверить accuracy, confusion matrix.  

### Январь — расширение моделей  
- Обучить Naive Bayes и Random Forest.  
- Сравнить все три модели по метрикам.  
- Сохранить лучшие результаты.  

### Февраль — апрель (продуктовые задачи)  
- Финальная модель уже должна быть выбрана (MVP).  
- Дальше работаем над продуктовой частью:  
  - как модель запускать и тестировать,  
  - сделать простое приложение (например, веб-форма: вставляешь текст → получаешь результат),  
  - настроить CI/CD.
